# PROJET 2.3 : Menu de Restaurant Interactif AVANCÉ

## Description

Ce projet a été réalisé dans le cadre de la formation **Akieni Academy – Phase 2 (HTML, CSS & Git)**.

L'objectif est de concevoir une page web présentant le menu d'un restaurant congolais en utilisant une structure HTML5 sémantique. La page met en avant différentes catégories de plats, leurs descriptions, leurs prix, ainsi que des informations pratiques comme les horaires d'ouverture et un formulaire de réservation.

---

## Objectifs pédagogiques

- Structurer une page complexe à l'aide des balises HTML5.
- Concevoir une navigation claire entre plusieurs sections.
- Présenter des données sous forme de tableaux.
- Créer un formulaire de réservation accessible.
- Appliquer les bonnes pratiques d'accessibilité et de sémantique HTML.

---

## Fonctionnalités

Le projet comprend les éléments suivants :

### Navigation

- Menu principal de navigation
- Navigation entre les différentes catégories du menu

### Menu du restaurant

Les plats sont organisés par catégories :

- Entrées
- Plats principaux
- Desserts
- Boissons

Chaque plat comporte :

- Nom
- Description
- Prix
- Liste des ingrédients
- Illustration

### Spécial du jour

Une section met en avant le plat recommandé du jour.

### Tableau des prix

Présentation d'un tableau récapitulatif permettant de comparer les prix par catégorie.

### Horaires d'ouverture

Affichage des horaires à l'aide d'une liste de définitions (`dl`).

### Réservation

Formulaire permettant à un client de réserver une table avec des champs correctement associés à leurs labels.

### Pied de page

Le footer contient les informations de contact du restaurant.

---

## Technologies utilisées

- HTML5
- Structure sémantique (`header`, `nav`, `main`, `section`, `article`, `aside`, `footer`)
- Formulaires HTML
- Tableaux HTML
- Images
- Bonnes pratiques d'accessibilité (`label`, `alt`, structure logique)

---

## Critères de validation

Le projet répond aux exigences suivantes :

- Utilisation d'une structure HTML5 sémantique complète.
- Navigation claire entre les différentes sections.
- Organisation des plats par catégories.
- Présence d'images illustrant les plats.
- Tableau de comparaison des prix.
- Utilisation d'une liste de définitions (`dl`) pour les horaires.
- Formulaire de réservation correctement structuré et labellisé.
- Footer contenant les informations de contact.
- Respect des bonnes pratiques d'accessibilité.
